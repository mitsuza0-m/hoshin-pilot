💻 Claude Code での編集ガイド
ファイル構成
各画面は独立した .html ファイルです。00_index.html がこの一覧です。番号順（01〜12）で画面の流れを把握できます。
カラー変数の変更
各ファイル冒頭の :root { ... } 内のCSS変数を変更すると全体の色が一括変更されます。--owner が経営者系、--amber が社員系の基本色です。
AI対話ロジックの場所
03_policy_editor.html 末尾の <script> に vSendMsg() / vGetResponse() 関数があります。AIのレスポンスパターンを編集して応答をカスタマイズできます。
データ管理（単一情報源）
日報・目標データは data/*.js が唯一の正データです。編集内容は HoshinStore（localStorage、キー接頭辞 hoshin_v8:）に保存され、同じブラウザで開いた他ページにも反映されます。一覧・入力フォーム・目標テーブルの共通ロジックは shared/*.js に分離されています。
SaaS化への拡張
各画面の toast() 関数でページ遷移を示しています。実装時はルーターの navigate() に置き換えることで画面遷移を実装できます。
